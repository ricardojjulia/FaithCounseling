# v5.7.0 Release Summary

**Version:** 5.7.0
**Date:** April 4, 2026
**Branch:** feat/workspace-studio-tabs-docs → main

---

## Summary

This release activates all five previously-placeholder tabs in Workspace Studio, completing the practice administration hub. Every tab that previously showed "Coming soon" is now fully functional. The release also adds cross-surface navigation shortcuts (Assign Forms from client portal, View Client from approved portal requests) and adds `onOpenCounselorMaintenance` wiring from App.jsx into the Staff tab.

---

## New Features

### Workspace Studio — Practice Tab

Edit the practice profile from inside Workspace Studio. Fields: practice name, type (solo/group/multi-location), timezone (US zones), faith tradition, contact email, and contact phone. Dirty-state badge and Save button; multi-practice support shows a switcher when more than one practice record exists.

API: `GET /api/v1/practices`, `PATCH /api/v1/practices/:id`

### Workspace Studio — Locations Tab

Full CRUD for scheduling locations. Add via modal, edit inline, delete via icon action. Fields: name (required), address, capacity (numeric), telehealth/remote-enabled toggle. Active locations show a Telehealth badge when remote-enabled.

API: `GET /api/v1/locations`, `POST /api/v1/locations`, `PATCH /api/v1/locations/:id`, `DELETE /api/v1/locations/:id`

### Workspace Studio — Staff Tab

Read-only staff roster loaded from the staff API. Counselors shown in a 2-column card grid with avatar (initials), role badge, license type, license number, supervision status, and bio excerpt. Admins shown in a flat list with last-login timestamp and account-locked indicator. "Manage Staff Accounts →" button navigates to the full Staff Management page.

API: `GET /api/v1/staff`

### Workspace Studio — Lifecycle Tab

Caseload management board. Four clickable status summary cards (Active, Waitlist, Inactive, Discharged) show counts with progress bars and act as filters for the client list below. A referral sources breakdown shows the top 8 sources with bar visualization. Each client row shows current status badge, high-touchpoint indicator, referral source, and a dropdown to move the client to any other status. Transitioning to "discharged" opens a modal to record reason (6 options: Treatment Complete, Client Request, Moved/Relocated, Referred Out, Non-Engagement, Other) and optional notes.

API: `GET /api/v1/clients?limit=500`, `PATCH /api/v1/clients/:id/lifecycle`

### Workspace Studio — Appointments Tab

Service code configuration for CPT/billing codes. Active/inactive/all filter pills. Add via modal, inline edit, activate/deactivate toggle. Fields: code (uppercased, max 30 chars), service name, category (Therapy, Assessment, Consultation, Group, Crisis, Other), default session duration (15–240 min in 5-min steps).

API: `GET /api/v1/billing/service-codes`, `POST /api/v1/billing/service-codes`, `PATCH /api/v1/billing/service-codes` (with `serviceCodeId`)

### Assign Forms Shortcut from Client Portal

The "Assigned Forms and Intake Packets" section in ClientPortalPage now has an "+ Assign Forms" button (visible to staff only via `onAssignForms` prop). Clicking it navigates directly to Workspace Studio → Documents tab with the current client pre-selected. Eliminates the need to manually switch surfaces and reselect the client.

### View Client on Approved Portal Requests

Approved portal requests that have a linked client (via `convertedClientId`) now show a "View Client" button in the Workspace Studio Portal tab. Previously, approved requests were a dead end with no navigation path to the converted client record.
