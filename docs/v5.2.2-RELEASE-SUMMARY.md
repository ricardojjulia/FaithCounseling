# v5.2.2 Release Summary — Offerings Settings and Removal Fixes

**Date:** March 30, 2026
**Branch:** `main`
**Release type:** Patch

---

## Overview

v5.2.2 completes the follow-on stabilization work for the new Offerings surfaces shipped in `v5.2.0` and hotfixed in `v5.2.1`.

This patch fixes the remaining real-world issues found while using Offerings end to end:

- existing databases could save portal settings reads but failed on write because the new offerings columns were never added to older `portal_settings` tables
- the Workspace Studio Offerings amount field could reload a saved cent value as dollars and drift on repeated saves
- the client portal giving view rendered `suggestedOfferingCents` as dollars instead of cents
- incorrect offering entries could be created, but there was no way to remove them from the Offerings workspace

---

## What Changed

### Database and API

- `apps/api/src/db/migrate.js`
  - adds backward-compatible `portal_settings` migrations for:
    - `financial_mode`
    - `suggested_offering_cents`
    - `offering_ministry_note`
- `apps/api/src/index.js`
  - fixes offerings runtime handling for authenticated requests
  - adds `DELETE /v1/offerings/:id` so staff can remove incorrect offering entries

### Workspace Studio and Portal

- `apps/web/src/components/WorkspaceStudio/tabs/OfferingsTab.jsx`
  - converts saved cent values back into dollars when loading the NumberInput
  - keeps the saved draft aligned after save so the amount is not multiplied on re-save
- `apps/web/src/components/Portal/ClientPortalPage.jsx`
  - formats `suggestedOfferingCents` and offering-history amounts as cent-based values
  - keeps already-normalized balance totals on the existing dollar formatter

### Offerings Workspace

- `apps/web/src/components/Offerings/OfferingsPage.jsx`
  - adds a `Remove` action to offering history entries
  - confirms deletion before removing an entry
  - reloads the offering history and summary after removal

### Documentation and versioning

- `README.md`
  - updates the current version and release focus
- `docs/change-log.md`
  - records the patch details and validation
- workspace package manifests
  - bump version from `5.2.1` to `5.2.2`

---

## Outcome

After this patch:

- the suggested offering setting saves correctly on older databases
- `$25.00` remains `$25.00` across Workspace Studio and the client portal
- the client portal giving surface no longer shows cent values as thousand-dollar amounts
- incorrect offering entries can be removed from the Offerings history
