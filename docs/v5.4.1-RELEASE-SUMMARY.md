# v5.4.1 Release Summary — Operations Studio Full Repair

**Date:** March 31, 2026
**Branch:** `main`
**Release type:** Patch
**Status:** ✅ Validated

---

## Overview

Operations Studio (`/operations.html`) appeared broken across all tabs — data was either missing or rendered incorrectly — despite the backend routes and handlers being fully implemented and wired to the database. The root cause was a collection of field-name mismatches accumulated between the API implementation and the standalone JavaScript frontend, plus one missing route for ending impersonation sessions.

This patch corrects all mismatches and fills the routing gap. No database migrations and no schema changes are required.

---

## Root Cause Analysis

### Why it looked like a placeholder

All seven Operations Studio API endpoints existed and were wired to the database with in-memory fallbacks:

| Endpoint | Handler |
| --- | --- |
| `GET /v1/reporting/overview` | `buildReportingOverview` |
| `GET /v1/audit/intelligence` | `buildAuditIntelligence` |
| `GET /v1/platform/overview` | `buildPlatformOverview` |
| `GET /v1/platform/tenant-provisioning` | `handleTenantProvisioning` |
| `GET/POST /v1/platform/impersonation-sessions` | `handleSupportImpersonationSessions` |
| `GET/POST /v1/platform/data-exports` | `handleDataExportJobs` |
| `GET/POST /v1/platform/retention-policies` | `handleRetentionPolicies` |

The API responses were correct. The frontend renderer functions (`renderPracticeReport`, `renderPlatformSummary`) referenced field names that no longer matched the API's actual response shapes — so every section silently rendered as blank or showed JavaScript errors.

### Specific mismatches found

**Reporting tab — `renderPracticeReport`:**

| Frontend referenced | API actually returns | Field |
| --- | --- | --- |
| `u.totalSessions` | `u.sessionsInWindow` | Utilization: session count |
| `u.completedSessions` | `u.sessionsCompleted` | Utilization: completed sessions |
| `u.remoteSessionRate` | `u.remoteRate` | Utilization: remote rate (already %) |
| `{ source, count }` | `{ referralSource, count }` | Referral source objects |
| `a.type` / `a.count` | `a.inventoryName` / `a.completedCount` | Assessment trend objects |
| *(flat fields)* | `{ requiresSignatureCount, signedDocuments, completionRate }` | Document completion shape |
| `thirtyToSixty` / `sixtyToNinety` / `ninetyPlus` | `days1to30` / `days31to60` / `days61to90` / `over90` | AR aging buckets (3 → 5) |
| `l.locationId` | `l.locationName` | Location display name |

**Platform tab — `renderPlatformSummary`:**

| Frontend referenced | API actually returns | Section |
| --- | --- | --- |
| `r.tenantId` | `r.requestedTenantId` | Provisioning table |
| `r.practiceName` | `r.requestedPracticeName` | Provisioning table |
| `r.role` | `r.targetRole` | Impersonation table |
| *(missing fallback)* | `endedAt − startedAt` | Impersonation duration when `durationMinutes` is null |
| `r.type` | `r.exportType` | Exports table |
| `r.requestedBy` | `r.requestedByRole` | Exports table |
| *(iterated as array)* | Object: `{ clinicalRecordsSchedule, billingSchedule, auditLogSchedule, … }` | Retention policy display |

**Tenant provisioning form — body payload:**

| Sent | Expected |
| --- | --- |
| `tenantId` | `requestedTenantId` |
| `practiceName` | `requestedPracticeName` |

**Retention policy save:**

| Issue | Fix |
| --- | --- |
| HTTP method `PUT` | API only accepts `POST` |
| `clinicalRecords` | `clinicalRecordsSchedule` |
| `billing` | `billingSchedule` |
| `auditLog` | `auditLogSchedule` |

**End impersonation session — missing route:**

The "End Session" button sent `PATCH /v1/platform/impersonation-sessions/:id` with the session ID in the URL path. The router only matched the collection URL (`/v1/platform/impersonation-sessions`) and had no handler for individual session IDs.

## What Changed

### `apps/web/public/operations.js`

**`renderPracticeReport`**

- All 8 field name mismatches corrected (utilization, referrals, assessments, document completion, AR aging, location)
- AR aging section expanded from 3 buckets to 5: `current`, `days1to30`, `days31to60`, `days61to90`, `over90`
- Document completion now reads the nested shape (`requiresSignatureCount`, `signedDocuments`, `completionRate`) instead of flat fields

**`renderPlatformSummary`**

- Provisioning table uses `requestedTenantId` and `requestedPracticeName` (with fallbacks for legacy data)
- Impersonation table uses `targetRole`; duration computed from `endedAt − startedAt` when `durationMinutes` is null
- Exports table uses `exportType` and `requestedByRole` (with fallbacks)
- Retention section completely rewritten: renders each known schedule field (`clinicalRecordsSchedule`, `billingSchedule`, `auditLogSchedule`) as a labeled pill with a human-readable value; shows legal hold enabled/disabled pill

#### Tenant provisioning form handler

- POST body corrected: `requestedTenantId`, `requestedPracticeName`

#### Retention policy save handler

- HTTP method changed from `PUT` to `POST`
- Field names corrected to `clinicalRecordsSchedule`, `billingSchedule`, `auditLogSchedule`

### `apps/api/src/index.js`

**New route — `PATCH /v1/platform/impersonation-sessions/:id`**

- Dispatches to `handleSupportImpersonationSessionById`
- Placed after the collection route, before the data-exports route

**New handler — `handleSupportImpersonationSessionById`**

- Extracts session ID from URL path (`pathname.split('/').pop()`)
- Handles `PATCH` only; other methods return 405
- DB path: calls `endImpersonationSession(sessionId)` and returns the updated row
- In-memory fallback: finds the session by ID, sets `status = 'ended'` and `endedAt` timestamp
- Returns appropriate 404 when session ID is not found

**`normalizePathname` update**

- Registered `/v1/platform/impersonation-sessions/:id` so OTEL traces normalize the parameterized path instead of logging raw session IDs
