# v5.0.0 Release Summary

## Operations Dashboard + Client Workspace Separation

**Release Date:** March 30, 2026  
**Version:** 5.0.0  
**Status:** Beta Ready  
**Scope:** Operations dashboard maturity plus restored client-maintenance workflows, clearer monitor issue-state reporting, and workspace-specific top-bar titles

## What Is Included

### Operations Dashboard

- real operations summary for staff users instead of placeholder cards
- counselor workload and 1-hour gap visibility for Today's Schedule
- Priority Queue backed by the real `highTouchpoint` client flag
- Compliance Watch metrics for unresolved note gaps and incomplete assigned work
- Clients summary card focused on totals, unscheduled clients, and portal request statuses
- dashboard drill-down workflows into client, scheduling, document, and portal-review queues
- alert thresholds for unscheduled high-touchpoint clients, documentation backlog, exhausted counselor capacity, and portal backlog
- 7-day trend views for utilization, documentation, portal flow, and unscheduled-client backlog

### Restored Clients Workspace

- dedicated `Clients` navigation surface instead of reusing the dashboard grid
- client lookup and status filtering
- client summary counts for active, waitlist, and high-touchpoint populations
- lightweight `Quick edit` modal for simple updates
- existing-client `Edit` restored to the full detailed client record

### Detailed Client Record

The detailed client-maintenance workflow is again the primary edit path for existing clients. Staff can now reach the fuller multi-tab record directly from the Clients workspace:

- demographics
- insurance
- contacts
- clinical
- diagnoses
- faith
- legal

## Features and Fixes

### Features

- added a real client-maintenance workspace for staff
- restored direct handoff from the Clients screen into chart and scheduling workflows
- restored full detailed client editing from the main Clients surface
- updated top bar copy so staff can distinguish the Clients workspace from the Operations Dashboard
- updated the top bar so the active workspace title changes for users, counselors, scheduling, documents, workspace studio, portal, and other main surfaces
- updated the monitor so surface failures distinguish current, recent, and historical issue state instead of only showing cumulative totals

### Fixes

- fixed the regression where `Clients` and `Dashboard` rendered nearly identical screens
- fixed the regression where existing-client `Edit` no longer opened the detailed client information screen
- fixed the monitoring ambiguity where one transient fetch failure could keep reading like a current outage
- fixed the top-bar fallback that labeled non-dashboard workspaces as `Operations Dashboard`
- fixed browser regression coverage so the restored Clients workspace and detailed edit path are both protected by Playwright
