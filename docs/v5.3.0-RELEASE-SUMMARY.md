# v5.3.0 Release Summary — Clinical Chart

**Date:** March 30, 2026
**Branch:** `main`
**Release type:** Minor
**Status:** ⚠️ UNTESTED — Under Review

---

## Overview

v5.3.0 ships the Clinical Chart — the primary clinical documentation surface for FaithCounseling. This release wires the "Clinical Chart" sidebar nav item (previously a placeholder) to a fully functional page with five clinical tabs: Session Notes, Internal Notes, Treatment Plan, Progress Tracking, and Homework.

The underlying database schema, API route handlers, and PHI encryption layer were already in place from prior releases. This release delivers the missing UI layer, one new API route (PATCH for note updates and sign/lock), and a new domain type (`internal_note`).

---

## What Changed

### Domain

- `packages/domain/src/index.js`
  - added `'internal_note'` to `progressNoteTypes` enum — a private counselor-only note type that is never locked and excluded from legal clinical record exports

### API

- `apps/api/src/index.js`
  - added route: `PATCH /v1/clients/:id/progress-notes/:noteId`
    - updates draft note content (summary, interventions, noteType)
    - signs and locks a note when `locked: true` is passed — sets `signed_by`, `signed_at`, and enforces immutability (409 on any subsequent mutation attempt)
    - enforces lock at the API layer, not just the UI
    - emits distinct audit events: `chart.progress_note.update` and `chart.progress_note.sign`

### Internationalization

- `packages/i18n/src/index.js`
  - added ~45 new keys across all clinical chart surfaces:
    - `chart.tab.*` — tab labels
    - `chart.note.*` — note composer, type labels, draft/lock states
    - `chart.plan.*` — treatment plan fields, status labels, placeholders
    - `chart.progress.*` — assessment history labels
    - `chart.homework.*` — homework assignment labels

### New UI Components

- `apps/web/src/components/ClinicalChart/ClinicalChartPage.jsx`
  - top-level page shell: client selector + 5-tab chart layout
  - client selection required before tabs activate

- `apps/web/src/components/ClinicalChart/tabs/SessionNotesTab.jsx`
  - create session notes with type, summary, and interventions
  - draft notes are editable inline
  - Sign & Lock flow: confirmation prompt → PATCH with `locked: true` → note becomes read-only with signed timestamp
  - note types: Intake Note, Progress Note, Treatment Plan Review, Discharge Note
  - internal notes filtered out of this tab

- `apps/web/src/components/ClinicalChart/tabs/InternalNotesTab.jsx`
  - private counselor notes (`note_type: internal_note`) — never locked, always editable
  - tags: Risk, Family dynamics, Spiritual concerns, Therapeutic progress, Collateral contact
  - displayed separately from clinical record notes

- `apps/web/src/components/ClinicalChart/tabs/TreatmentPlanTab.jsx`
  - goals editor: dynamic add/remove/edit list with per-goal textarea
  - interventions editor: same pattern
  - status selector (Draft / Active / On Hold / Completed)
  - review cadence and last-reviewed date fields
  - saves via existing `PUT /v1/clients/:id/treatment-plan` endpoint
  - dirty-state tracking to prevent accidental no-op saves

- `apps/web/src/components/ClinicalChart/tabs/ProgressTab.jsx`
  - pulls all form submissions for the client from `/v1/forms/submissions`
  - groups scored instruments by form title
  - shows per-instrument score history (newest first) with severity band badges
  - severity bands for: PHQ-9, GAD-7, BAI, PCL-5, OCI-R, AUDIT, ISI, RSES, DASS-21, ACE
  - unscored assessments shown in a separate all-submissions table with submitted-by type

- `apps/web/src/components/ClinicalChart/tabs/HomeworkTab.jsx`
  - pulls form assignments from `/v1/forms/client-overview` (falls back to `/v1/forms/assignments`)
  - pending assignments shown as cards with status badge, timing type, counselor notes, and due date
  - completed assignments shown in a summary table
  - cancelled assignments summarized by count, not shown in detail

### Routing

- `apps/web/src/App.jsx`
  - imported `ClinicalChartPage`
  - added `showClinical` flag (`currentView === 'clinical'`)
  - updated `showFallbackWorkspace` to exclude `showClinical`
  - removed `'clinical'` from telemetry `emptyState: 'placeholder'` list — it is now a real surface
  - `showClinical` renders `<ClinicalChartPage clients={clientsData.items} currentUser={currentUser} />`

## Data / Compliance Notes

- All session note and treatment plan content remains encrypted at rest via the existing `encrypt()` / `decrypt()` layer. No schema changes were required.
- Internal notes (`note_type = 'internal_note'`) are stored in `progress_notes` with `locked = 0` always. They share the same encrypted storage and audit trail but are excluded from the session notes tab.
- Lock enforcement is at the API layer: `PATCH` on a locked note returns `409 Conflict` regardless of how the request is made.
- All mutations emit audit events to the existing audit pipeline.

## Out of Scope (Deferred)

- Rich text editor (TinyMCE / Tiptap) — `@mantine/tiptap` not in dependencies; Textarea used
- SOAP structured note body (subjective / objective / assessment / plan) — requires schema migration
- Per-goal progress scores over time — requires `treatment_plan_goal_progress` table
- Note version history viewer — requires `progress_note_versions` table
- Appointment linkage on session notes — requires `appointment_id` FK on `progress_notes`
- Homework: assign new forms from the chart tab (use Workspace Studio → Documents & Inventories)
