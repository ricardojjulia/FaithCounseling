# v5.2.0 Release Summary — Offerings Model

**Date:** March 30, 2026
**Branch:** `main`
**Commit:** follows v5.1.0 (`96339a8`)

---

## Overview

v5.2.0 replaces the generic billing model with a faith-based voluntary offering system throughout the application. The change reflects how Christian counseling ministries actually operate: clients give freely rather than pay invoices, no insurance is tracked in the UI, and the suggested offering amount is a pastoral guidance rather than a clinical charge. The insurance tab is soft-deprecated — the underlying schema data is preserved, but the UI surface is removed.

This is a full-stack change spanning i18n, surface registry, API schema, API routes, three new or rewritten React components, and updated governance documentation.

---

## What Changed

### Navigation and Naming

- The **Billing** sidebar entry is now **Offerings**.
- The **Financials** portal tab is now **Giving**.
- The **Workspace Studio Billing** tab is now **Offerings**.
- All i18n keys (`nav.billing`, `studio.tab.billing`, `portal.tab.financials`, `portal.financials.*`) have been renamed to their `offerings` and `giving` equivalents.
- All telemetry surface IDs have been updated in `packages/telemetry/src/surfaces.js` to match.

### Insurance Tab Removal

- The **Insurance** tab has been removed from the client detail record.
- The import, TABS array entry, surface map entry, and panel have all been removed from `ClientDetailTabs.jsx`.
- The underlying `client_insurance` DB table is retained; no data migration is needed.
- The `client.insurance` surface ID has been removed from the surface registry.

### Database Schema

- `portal_settings.financial_mode` default changed from `'billing'` to `'offerings'`.
- Two new columns on `portal_settings`: `suggested_offering_cents INT NOT NULL DEFAULT 0` and `offering_ministry_note TEXT NULL`.
- New `offerings` table: `id`, `tenant_id`, `client_id`, `counselor_id`, `amount_cents`, `received_on DATE`, `note`, `created_by`, `created_at`, indexed on `(tenant_id, client_id)` and `(tenant_id, received_on)`.

### API Routes

- `/v1/portal/settings` PATCH now handles `suggestedOfferingCents` and hard-locks `financialMode` to `'offerings'`.
- New route: `GET /v1/offerings` — list offerings (tenant-scoped, client-role restricted to own records).
- New route: `POST /v1/offerings` — record a new offering.
- New route: `GET /v1/offerings/summary` — aggregated stats: `thisMonthCents`, `yearToDateCents`, `totalCents`, `count`, `averageCents`.

### New Components

#### `OfferingsTab.jsx` — Workspace Studio

- Loads `GET /v1/portal/settings` and `GET /v1/offerings/summary` in parallel.
- NumberInput (dollar amount, `$` prefix) → PATCH `/v1/portal/settings` with `{ suggestedOfferingCents }`.
- Four summary cards: This month / Year to date / Total received / Average per session.
- Telemetry via `useSurfaceTelemetry('studio.offerings', ...)`.

#### `OfferingsPage.jsx` — Main Workspace

- Full top-level workspace for the **Offerings** nav item.
- Loads offerings list and summary in parallel.
- Summary cards + offering history list with client name, amount, date, note, and "received" badge.
- Record Offering modal: client Select (searchable), dollar NumberInput, date TextInput, note Textarea → POST `/v1/offerings`.
- Telemetry: `useSurfaceTelemetry('offerings', { surfaceKind: 'view', workflow: 'navigation' })`.

#### `ClientPortalPage.jsx` — Client Portal Giving Tab

- Renamed surface map entry: `financials` → `portal.giving`.
- Tab label updated to `portal.tab.giving`.
- Dashboard card always shows the suggested offering amount.
- Giving panel now reads the configurable ministry note from portal settings and renders offerings-only content: ministry note, suggested amount, three stat cards, and recent offerings list.

### Governance Documentation

- `PLANS/FULL-SURFACE-MONITORING.md` — surface inventory updated: `billing` → `offerings`, `portal.financials` → `portal.giving`, `client.insurance` removed, `studio.billing` → `studio.offerings`.
- `docs/domain-model.md` — Billing module renamed to Offerings; scheduling entities updated (Invoice/Superbill/Payment replaced by Offering entity and `PortalSettings.suggestedOfferingCents`); permissions table Billing column renamed to Offerings.
- `docs/PRODUCT-PLANS-OVERVIEW.md` — new file summarising all PLANS files (this release).
