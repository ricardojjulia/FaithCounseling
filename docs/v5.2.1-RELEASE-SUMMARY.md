# v5.2.1 Release Summary — Offerings UI Hotfix

**Date:** March 30, 2026
**Branch:** `main`
**Release type:** Patch

---

## Overview

v5.2.1 fixes the first-release regressions on the new Offerings surfaces shipped in v5.2.0.

Two issues were corrected:

- the React shell displayed raw translation keys such as `nav.offerings` and `topbar.offerings.title`
- the Offerings screens fetched `/v1/...` directly instead of `/api/v1/...`, causing the web server to return HTML and the UI to fail with `Unexpected token '<'`

This patch is intentionally narrow: it corrects the frontend message catalog and the frontend fetch paths, then republishes the built web assets.

---

## What Changed

### Frontend i18n

The missing Offerings and Giving strings were added to the actual frontend catalog in `packages/i18n/src/index.js`.

New keys include:

- `nav.offerings`
- `topbar.offerings.title`
- `topbar.offerings.subtitle`
- `studio.tab.offerings`
- `offerings.title`
- `offerings.subtitle`
- `offerings.suggestedLabel`
- `offerings.suggestedHelp`
- `offerings.saveSuggested`
- `offerings.summaryTitle`
- `offerings.offeringsTitle`
- `offerings.thisMonth`
- `offerings.yearToDate`
- `offerings.totalOfferings`
- `offerings.averageOffering`
- `offerings.noOfferings`
- `offerings.recordOffering`
- `offerings.ministryNoteLabel`
- `offerings.ministryNoteHelp`
- `portal.tab.giving`
- `portal.giving.*`

### Frontend fetch paths

The Offerings React surfaces now use the same-origin API proxy:

- `apps/web/src/components/Offerings/OfferingsPage.jsx`
- `apps/web/src/components/WorkspaceStudio/tabs/OfferingsTab.jsx`

The updated requests are:

- `/api/v1/offerings`
- `/api/v1/offerings/summary`
- `/api/v1/portal/settings`

### Built assets

The web bundle was rebuilt and `apps/web/public/index.html` now points at the updated production asset entry.

## Outcome

After this patch:

- the sidebar shows `Offerings` instead of `nav.offerings`
- the top bar shows the real Offerings title and subtitle
- the Offerings workspace and Workspace Studio Offerings tab load JSON correctly
- the `Unexpected token '<'` load failure is removed
